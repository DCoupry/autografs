# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

AuToGraFS generates Metal-Organic Frameworks (MOFs), COFs and other periodic crystalline structures by mapping molecular building blocks (SBUs) onto topological blueprints, and runs the pipeline in reverse (deconstruction). Python >= 3.11. Built on pymatgen, ASE, networkx, scipy. Docs: `README.md` is the overview + FAQ; feature guides live in `docs/*.md` (building, cofs-and-stacking, editing, deconstruction, cli, extending, internals, coverage). Keep them accurate to the current API тАФ when changing behaviour, update the relevant `docs/` page (and the README FAQ/Why bullets if the change is user-facing).

Console entry points: `autografs` (interactive wizard, `cli.py`) and `autografs-topologies` (CGD тЖТ JSON topology converter, `cgd.py`).

## Architecture

Modules in `src/autografs/`, in pipeline order:

- **`fragment.py`** тАФ `Fragment`: a pymatgen `Molecule` plus dummy-atom (`"X"`) connection points. Represents both SBUs and topology slots. Compatibility (`has_compatible_symmetry`) is geometric: directional RMSD between unit arm vectors (threshold `COMPATIBILITY_MAX_RMSD = 0.35`), not point-group symbols. Also pre-build editing: `rotate`, `flip`, `functionalize`.
- **`topology.py`** тАФ `Topology`: a pymatgen `Lattice` plus an array of `Fragment` slots. `mappings` groups slots by crystallographic orbit (`equivalence_class`); one SBU choice covers all slots of an orbit. `is_2d` marks layer nets (c is frozen slab padding).
- **`topology_io.py`** тАФ versioned JSON (de)serialization of topology libraries; `LazyTopologyLibrary` materializes entries on first access (don't `dict.update()` it тАФ that materializes everything). Lookup-only aliases via `attach_aliases` (the builder attaches `data/iza_aliases.json`, 55 IZA zeolite codes тЖТ lowercase RCSR names; aliases resolve in `[]`/`in` but never appear in iteration). Legacy dill pickles still load with a warning.
- **`alignment.py`** тАФ the numpy build core. Directional matching (Hungarian + Kabsch, proper rotations only тАФ chirality preserved) and the cell objective (RMS deviation of anchor-pair distances from Cordero covalent bond lengths, over the crystal system's free parameters only). Everything precomputed into a `BuildPlan`; no pymatgen objects inside the optimization loop.
- **`builder.py`** тАФ `Autografs`: entry point. Loads SBUs (`data/defaults.xyz` + `data/pormake.xyz` + user XYZ) and topologies (`data/topologies.json.gz`). `build(topology, mappings, max_rmsd=..., min_distance=...)` тЖТ `Framework`; gates raise `AlignmentError`/`OverlapError` instead of returning bad structures. `build_all()` enumerates combinations (seeded sampling above `max_per_topology`, `n_jobs` worker processes). Module-level `build_framework()` is the library-independent core used by workers.
- **`framework.py`** тАФ `Framework`: the result. The networkx bond graph is the source of truth (symbols, coords, bond orders, UFF4MOF types, tags, per-atom `slot`/`sbu` provenance); views/exports: `structure`, `to_ase`, `write_cif`, `to_gulp`, `min_contact`, `stack` (2D layer тЖТ COF crystal, AA/AB/serrated/staggered), `relax` (in-process LAMMPS/UFF4MOF, optional `[relax]` extra). Post-build editing methods (`slots`, `supercell`, `defects`, `rotate`, `flip`, `functionalize`) are thin delegates into `editing.py`.
- **`framework_io.py`** тАФ versioned JSON (de)serialization of Framework objects (`Framework.save`/`load`): persists what CIF loses тАФ the bond graph with orders, UFF4MOF types, anchor tags, and slot/SBU provenance тАФ so a framework saved in one session stays editable in another.
- **`porosity.py`** тАФ geometric porosity descriptors behind `Framework.void_fraction`/`largest_cavity_diameter`/`pore_limiting_diameter`: a periodic vdW-surface distance grid; PLD by binary search over probe radius with a wrap-detecting union-find (a percolating channel = an open component connected to its own periodic image).
- **`net.py`** тАФ labeled quotient graphs (one node per slot/unit, edges carry periodic-image voltages). Verification (`verify_net`: exact edge-multiset comparison, possible because built atoms record slot provenance; **rod** frameworks тАФ whose slots are repeats+linker placements, not blueprint slots тАФ instead compare the framework's uncontracted signature to the blueprint's PoE form `topology_rod_quotient_edges` over every axial/helical run, the gcd fold absorbing the `n_repeats` supercell, `build_rod(..., verify_net=True)` gating it, #158) and identification (`identify_net`: coordination-sequence signatures, caps pruned / 2-c vertices contracted, matched two-tier тАФ uncontracted first, so a net beats its edge-decorated derivatives; a degree-multiset prefilter reads `raw_items()` and scans the full library without materializing it). `axial_runs` (rod Stage C2) finds a blueprint's *straight* slot runs тАФ cyclic quotient walks with every step displacement parallel and a nonzero closure voltage (pcu: three node+axial-edge chains; zigzag nets: none). `helical_runs` (#158) finds the *spiralling* runs straight detection misses тАФ a constant-axial-advance walk whose node slots fit a constant screw about a fixed axis line (`_fit_screw`): reports `HelicalRun` with signed `screw_angle`/`screw_order` (the blueprint mirror of a rod's own `RodRepeat` screw), so etb resolves to its 3_1 metal-oxo channels (order 3, ┬▒120┬░) and srs to its 4_1 (order 4, +90┬░); pcu/zigzag give none, and 2D layer nets are skipped (an in-plane zig-zag is no 3D rod channel).
- **`deconstruct.py`** тАФ the inverse pipeline. CIF/Structure тЖТ EconNN bonds capped at `BOND_LENGTH_SLACK` ├Ч UFF radius sums (EconNN is adaptive and bonds isolated guests to anything) тЖТ 0-periodic components removed as guests тЖТ clustering тЖТ dummy-capped `Fragment`s (X at cut-bond midpoints) тЖТ quotient graph (built directly from the cut list: one voltage-labeled edge per cut bond, so parallel bonds through different periodic images all survive; gauge shared with `autografs.net`) тЖТ `identify_net`. Guest removal iterates until the bond graph is guest-free, so the fold and the subframework split come from one final graph. Clustering dispatches on metal presence: **metal-oxo** for MOFs (metals + C-free metal-bound atoms + carboxylate/sulfonate/phosphonate groups тАФ cuts at carboxylate-CтЖФbackbone-C and metalтЖФdonor bonds), **branch-point** for metal-free COFs (`_organic_units`: ring systems + non-ring atoms collapse to super-vertices via `_ring_bonds` тАФ a bounded zero-voltage cycle search on the periodic graph, no global unwrap тАФ and external-degree тЙе3/2/1 тЖТ node/linker/cap; single-node convention). Rod (1-periodic) units are detected and reported as `RodUnit`s (axis + repeat from the rank-1 mismatch lattice) and enter the quotient as points of extension joined along the axis (O'Keeffe PoE convention; typically a contracted-tier match тАФ no edge centers); bare rods with no cuts and 2-periodic (layer) units raise `DeconstructionError`. Interpenetrated structures split into per-periodic-component quotient graphs, each identified independently (`subframework_nets`); `net_candidates` is their consensus, `n_periodic_components` the fold. Entry points: `Autografs.deconstruct(source)`, module-level `deconstruct(source, topologies=...)`.
- **`fingerprint.py`** тАФ canonical assembly fingerprints: `AssemblyFingerprint` = (nets, block multiset gcd-reduced, fold), hashable, so "is this (net, SBU) combination realized?" is a set lookup. `from_deconstruction(result, library=...)` re-expresses blocks in a harvest's vocabulary via `deconstruct.match_fragment` (the read-only cousin of `merge_fragment`; unmatched blocks get an `unmatched:` marker that never collides); `from_framework(framework)` reads per-slot `sbu` provenance directly тАФ the two meet when frameworks are built *from* the harvested library. Rod deconstructions have no finite block multiset and raise ValueError.
- **`harvest.py`** тАФ batch SBU harvesting: runs `deconstruct` over many structures (directory/glob/iterable) and merges building units into one cross-structure-deduplicated `Fragment` library (`merge_fragment` from `deconstruct.py`), keeping per-fragment provenance, per-source net candidates, and a non-aborting failure report. `HarvestResult.building_units` excludes monotopic `cap` fragments (bound solvent). Rods dedupe separately into `HarvestResult.rods`/`rod_provenance` via `rods.merge_rod`. Entry: `Autografs.harvest(sources)`.
- **`rods.py`** тАФ canonical rod identity behind harvest rod dedup (rod Stage B, #91): `canonical_rod(structure, rod_unit)` тЖТ `RodRepeat`, one *chemical* repeat in a cylindrical frame (axis line through the perpendicular centroid тАФ the only transverse origin a screw symmetry allows). Screw detection by self-matching under (axial shift, rotation): crystallographic repeat = screw_order ├Ч chemical repeat (pitfall 3), falls back to no reduction when the cut pattern breaks the atoms' screw symmetry. `matches` compares modulo rotation ├Ч phase ├Ч proper flip (Hungarian per element in the cylindrical metric); **signed** screw angle is part of the identity тАФ it is frame-invariant once the axial step is normalized positive, and enantiomeric helices must not merge (proper flips preserve helicity; don't "fix" this to abs()). Stage C1: `rod_fragment` тЖТ `RodFragment` (the buildable form: local-frame atom template + connection arms from `RodUnit.cut_vectors` + internal bond graph from `RodUnit.internal_bonds`, both rotated/mapped into the cylindrical template frame). `_local_positions` unwraps the rod by walking its bond graph **intra-cell bonds first** (periodic-closure bonds last), so a screw rod lays out monotonically along its axis тАФ the chemical-repeat reduction then keeps naturally-bonded atoms together and the **screw-aware** slab correspondence (de-screw azimuth by slab) records correct template bonds for **helical** rods too, not just straight (#158). `STRAIGHT_SCREW_TOL` still gates forward *building* (`build_rod` rejects helical placement тАФ that's the next #158 increment), but the fragment itself is now complete for helical rods. Versioned JSON sidecar via `save_rods`/`load_rods` (`HarvestResult.write_rods`) тАФ rods can't join the XYZ SBU format.
- **`rod_build.py`** тАФ forward building of rod frameworks, straight **and helical** (rod Stage C) behind `Autografs.build_rod` / `build_rod_framework(topology, rod, linkers)`: places a `RodFragment` on a blueprint's slot run + finite SBUs on every other slot. Two run kinds: a **straight** run (`net.axial_runs`, one PoE node/period, supercelled `n_repeats` up) or a **helical** run (`net.helical_runs`, `screw_order` node slots filled 1:1 by the rod's repeats тАФ no supercell; `_select_run` matches the rod's signed screw, #158). The rod **pins** the run-axis cell parameter to `n_repeats ├Ч chemical repeat` (removed from the free set; `n_repeats = max(2, screw_order)` тАФ enough to close the screw, тЙе2 so the continuation bond is a distinct node pair, not a parallel edge тАФ pitfall 9); placement lays down `n_repeats` copies by the **screw operation** (copy *n* rotated `n ├Ч screw_angle` about the axis тАФ about the **run's axis line** `axis_point` for a helical run, the node itself for straight; linkers spiralling with it тАФ pure translation for a straight rod), optimizing (in-plane scale, axis rotation, axial phase) against covalent anchor-length targets (pitfall 10); a second **relief** pass rotates each ditopic linker about its own arm axis (keeps anchors fixed тЖТ bonds stay closed) to maximize the minimum inter-unit contact, one shared ╧Ж per orbit (un-clashes coplanar rings тАФ the finite pipeline leaves that to manual `rotate()`). Rod-linker bonds are **explicit graph edges** (tags assume one connection/atom; a rod anchor carries several), all atoms tag 0. Closure + RMSD + `min_contact` gates. **Cross-linked multi-rod nets** (`etb` / MOF-74 proper, #158): `_select_runs` returns **every** matching helical run (etb: six interleaved 3_1 helices, three of each hand); `_RodBuild` holds one `rod_specs` entry per run and places a rod on each тАФ reflecting the template/arms (local yтЖТтИТy) for opposite-handedness runs since etb is centrosymmetric тАФ with the linkers on the slots no run claims (they bridge *different* rods). The reference node per run is **unwrapped** onto its own cylinder (`_unwrapped_node_frac`; a home-cell image can be a perp-wrapped copy off the axis, breaking radius/azimuth). `net._verify_rod_net` groups helical runs by `screw_order` and contracts each group's edge-centers (`topology_rod_quotient_edges` takes a run list), so etb's full PoE form is checked. **Mixed rod/finite mappings** (#168): `linkers` is a Fragment *or* a `{slot type | slot index: SBU}` mapping (`_validate_linkers`, covering the lateral slots only тАФ the run is the rod's), lateral SBUs may be **polytopic**, and lateral arm counts come from **run membership** (`_slot_adjacency`: node degree тИТ the connections the run consumes) rather than the old `degree тИТ 2`. Placement reuses `alignment.match_directions` + `kabsch` (Hungarian arm assignment, re-solved once at the optimized cell by `refresh_assignments` тАФ a polytopic slot's arm directions move with the anisotropic cell); relief ╧Ж applies only to species with a genuine arm axis (2 near-antiparallel arms). The old bipartite rod-armтЖФlinker-anchor Hungarian is replaced by `_pair_ports`: **every** connection point (rod arm tip + every SBU dummy) pairs with a nearest port of another unit, greedily by min-image distance тАФ no library net with a rod run has polytopic laterals bonded only to run nodes, so linkerтЖФlinker bonds are unavoidable. Greedy carries a **feasibility guard** (ports form a complete multipartite graph тЖТ a perfect matching exists iff no unit holds more than half the free ports; checked per candidate), which is not just a safety net: a rod carrying half the ports *forces* every bond to be a rod bond, which is what keeps single-helix nets like `unc` correct. Validated end-to-end: synthetic 2_1 helix on pcu; **4_1 тАУZnтАУOтАУ helix on `unc`** (single-helix, builds clean, `verify_net(unc)` passes, round-trips to net `unc` + a screw-order-4/+90┬░ rod); **six cross-linked 3_1 rods on `etb`** (`verify_net(etb)` passes, `identify_net` тЖТ `etb`, both handednesses placed; the small cell crowds the synthetic fixture so the test relaxes `min_distance` тАФ `relax()` cleans it); and **`etb-e` with two lateral species** (36 ditopic + 9 tetratopic laterals, 36 linkerтЖФlinker bonds, `verify_net` + `identify_net` тЖТ `etb-e`, and it clears the *default* contact gate unrelaxed тАФ the MOF-74 bonding as it really is). Scope: single-axis runs; multi-axis (woven) packings and multi-PoE-per-run тАФ future work.
- **`editing.py`** тАФ post-build editing on the bond graph: exact supercells (per-edge periodic image shift recovered by min-image rounding of unwrapped coords), statistical defects (seeded SBU removal + capping of dangling anchors), placed-SBU rotation/flip (isometries that fix the anchor atoms), framework functionalization (terminal-atom replacement by X-tagged groups). All ops return new Frameworks. The tag/anchor-based ops (`defects`/`rotate`/`flip`/`functionalize`) refuse rod (`build_rod`) frameworks via `_reject_rod` (rod Stage C4, #91): rods carry explicit inter-unit bonds and no anchor tags, and a rod has no finite SBU to remove/reorient (pitfall 16); `Framework.is_rod` reads the `rod_build` graph marker, which `replicated_graph` and `framework_io` propagate so supercell and save/load keep the guard.
- **`charges.py`** тАФ partial charges behind `Framework.assign_charges`: EQeq (Wilmer et al. 2012) as a numpy port validated against the reference C++ (element tables in `data/eqeq.py`; the exotic "Ewald" convention тАФ erfc(R/╬╖)-damped real sum + fixed-range reciprocal sum, ╬╖=50 ├Е тАФ is reproduced literally, don't "fix" it). `CHARGE_METHODS` registry for pluggable schemes. Charges live on graph nodes (`charge` attr, `charge_method` graph attr), survive `framework_io` and editing, and flow into exports: CIF `_atom_site_charge` (P1 only), ASE initial charges, GULP (drops `noelectrostatics`).
- **`relax.py`** тАФ LAMMPS relaxation backend behind `Framework.relax`. Shared staging helpers (`_write_lammps_inputs`, `_launch_lammps`) also serve `elastic.py`.
- **`ase_relax.py`** тАФ ASE-calculator bridge behind `Framework.relax(calculator=...)`: any ASE calculator (or by name: "gfn-ff" via xtb-python, "gfn1" via tblite, "dftb"; "gfn2" rejected тАФ no periodic implementation) drives a FrechetCellFilter+FIRE relaxation; bond graph preserved (order-canonical rebuild, #145), eV converted to the kcal/mol `Framework.energy` convention. All backends optional, import-guarded with install hints.
- **`elastic.py`** тАФ elastic constants behind `Framework.elastic_properties`: stress-strain central differences on the in-process LAMMPS session (LAMMPS ELASTIC-example protocol: relax, then per-Voigt-strain `change_box` + fixed-cell FIRE), point-group projection of the tensor (averaged over cartesian rotations from `SpacegroupAnalyzer`), VRH moduli + directional Young's extrema on `ElasticProperties`.
- **`twist.py`** тАФ commensurate twist search + moir├й bilayers behind `Framework.stack(mode="twisted", angle=...)`: plane-group-general CSL enumeration (coincidence vector pairs clustered by angle; reproduces the hexagonal closed form and the square `2┬╖atan(n/m)` family тАФ the tests use those as independent oracles), strained-commensurate mode (`max_strain` absorbs mismatch into the top layer, reported), snap-to-nearest with `angle_tolerance`, `max_atoms` guardrail (supercell ~1/╬╕┬▓). Family completeness is bounded by `max_index` тАФ a family is only found once *two* independent coincidence vectors fit the index range. Exact angle/strain recorded as `twist_angle`/`twist_strain` graph attrs.
- **`plane_groups.py`** тАФ the 17 plane groups, used for 2D layer nets.
- **`cgd.py`** тАФ CGD parser and `autografs-topologies` CLI (RCSR download, spacegroup expansion, orbit extraction; `--use_iza` fetches + converts the IZA zeolite CIFs via `fetch.py` + `extract_topology.py`, entries under official uppercase codes).
- **`fetch.py`** тАФ polite local fetching of externally-licensed sources (IZA now, EPINET later): license-acceptance gate (`require_acceptance`, interactive or `--accept-licenses`; nothing restricted ships in the wheel), resumable atomic cache (`~/.autografs/cache/<source>`), identifying UA, fixed request delay. Code list + filename convention (prefixes `-`/`*` stripped server-side) in `data/iza_codes.py`, shared with `scripts/make_iza_aliases.py`.
- **`extract_topology.py`** тАФ `topology_from_tetrahedral(structure, name)`: idealized zeolite crystal (T atoms + bridging O, e.g. IZA CIFs) тЖТ buildable Topology. Emits the exact CGD conventions (Z-encodes coordination on centers, X dummies at TтАУO midpoints = quarter points) and reuses `cgd.analyze`/`orbit_equivalence_classes`, so extracted zeolites are indistinguishable from CGD-imported nets down to the exact identification tier. Rejects interrupted/non-tetrahedral input (`TopologyExtractionError`). `topologies_from_tetrahedral_cifs` batches it with per-entry error accounting тАФ the `--use_iza` conversion step.
- **`cli.py`** тАФ `autografs` interactive wizard (questionary + rich). Deliberately has no non-interactive build flags; scripted use is the Python API.
- **`utils.py`** тАФ XYZ parsing, UFF4MOF atom typing (constants in `data/uff4mof.py`), FragmentтЖТgraph conversions, GULP export.
- **`exceptions.py`** тАФ `AutografsError` hierarchy: `AlignmentError`, `OverlapError`, `StackingError`, `RelaxationError`, `TopologyExtractionError`, `NetMismatchError`, `DeconstructionError`.

Pipeline: libraries тЖТ geometric compatibility sieve (`list_building_units(sieve=...)` / `list_topologies(sieve=...)`) тЖТ slot-type-to-SBU mappings тЖТ per-slot alignment + cell optimization тЖТ quality gates тЖТ `Framework`.

## Gotchas

- **Everything ships in the wheel**: `data/topologies.json.gz` (2686 RCSR nets) and both SBU files are bundled; no generation step is needed. Regeneration (only when updating the library): `autografs-topologies --use_rcsr -o topologies.json.gz`. `scripts/cgd2pkl.py` is a deprecated shim for that command.
- **Research tooling lives in `scripts/`, not the library**: `scripts/benchmarks/` (net-ID accuracy, round-trip closure, throughput), `scripts/funnel/` (staged multi-fidelity screening driver: level escalation asbuiltтЖТUFF4MOFтЖТGFN-FF/GFN1/DFTB+, top-N selection, per-candidate provenance, checkpoint/restart compatible with `build_all` checkpoint dirs, Spearman rank-stability report), and `scripts/rods/scan_rod_mofs.py` (deconstruct a directory of CIFs, report rod building units with canonical screw/net; `report --elements` palette filter is **optional, off by default** тАФ scans all; reads a user directory, redistributes nothing). All have path-insertion smoke tests in `tests/` so CI catches API breakage.
- Dummy atoms use pymatgen's `"X"` species convention throughout. Slot/SBU compatibility is decided by dummy count + arm-direction match, NOT point group (symbols are metadata only).
- In `cgd.py`, `# EDGE_CENTER` lines in RCSR files look commented out but are deliberately uncommented by the parser (they create the 2-connected slots). Preserve that behavior when refactoring.
- `Autografs._validate_mappings` deep-copies fragments: alignment mutates them in place, and aliasing would corrupt the SBU library. Keep that invariant.
- Framework node ids must stay contiguous 0..n-1: `min_contact` and relax index pymatgen Structure sites by sorted node id. Any graph-editing operation must relabel before returning (see `editing._relabel`). Graph node coords are unwrapped cartesian; a bond's periodic image is implicit and recovered by min-image rounding тАФ editing relies on it.
- Formatting/linting is ruff (not black/isort/flake8); mypy targets 3.12 even though the runtime floor is 3.11 (numpy stubs use PEP 695).
