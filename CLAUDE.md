# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

AuToGraFS generates Metal-Organic Frameworks (MOFs), COFs and other periodic crystalline structures by mapping molecular building blocks (SBUs) onto topological blueprints, and runs the pipeline in reverse (deconstruction). Python >= 3.11. Built on pymatgen, ASE, networkx, scipy. Docs: `README.md` is the overview + FAQ; feature guides live in `docs/*.md` (building, cofs-and-stacking, editing, deconstruction, cli, extending, internals, coverage). Keep them accurate to the current API â€” when changing behaviour, update the relevant `docs/` page (and the README FAQ/Why bullets if the change is user-facing).

## Architecture

Modules in `src/autografs/`, in pipeline order:

- **`fragment.py`** â€” `Fragment`: a pymatgen `Molecule` plus dummy-atom (`"X"`) connection points. Represents both SBUs and topology slots. Compatibility (`has_compatible_symmetry`) is geometric: directional RMSD between unit arm vectors (threshold `COMPATIBILITY_MAX_RMSD = 0.35`), not point-group symbols. Also pre-build editing: `rotate`, `flip`, `functionalize`.
- **`topology.py`** â€” `Topology`: a pymatgen `Lattice` plus an array of `Fragment` slots. `mappings` groups slots by crystallographic orbit (`equivalence_class`); one SBU choice covers all slots of an orbit. `is_2d` marks layer nets (c is frozen slab padding).
- **`topology_io.py`** â€” versioned JSON (de)serialization of topology libraries; `LazyTopologyLibrary` materializes entries on first access (don't `dict.update()` it â€” that materializes everything). Lookup-only aliases via `attach_aliases` (the builder attaches `data/iza_aliases.json`, 55 IZA zeolite codes â†’ lowercase RCSR names; aliases resolve in `[]`/`in` but never appear in iteration). Legacy dill pickles still load with a warning.
- **`alignment.py`** â€” the numpy build core. Directional matching (Hungarian + Kabsch, proper rotations only â€” chirality preserved) and the cell objective (RMS deviation of anchor-pair distances from Cordero covalent bond lengths, over the crystal system's free parameters only). Everything precomputed into a `BuildPlan`; no pymatgen objects inside the optimization loop.
- **`builder.py`** â€” `Autografs`: entry point. Loads SBUs (`data/defaults.xyz` + `data/pormake.xyz` + user XYZ) and topologies (`data/topologies.json.gz`). `build(topology, mappings, max_rmsd=..., min_distance=...)` â†’ `Framework`; gates raise `AlignmentError`/`OverlapError` instead of returning bad structures. `build_all()` enumerates combinations (seeded sampling above `max_per_topology`, `n_jobs` worker processes). Module-level `build_framework()` is the library-independent core used by workers.
- **`framework.py`** â€” `Framework`: the result. The networkx bond graph is the source of truth (symbols, coords, bond orders, UFF4MOF types, tags, per-atom `slot`/`sbu` provenance); views/exports: `structure`, `to_ase`, `write_cif`, `to_gulp`, `min_contact`, `stack` (2D layer â†’ COF crystal, AA/AB/serrated/staggered), `relax` (in-process LAMMPS/UFF4MOF, optional `[relax]` extra). Post-build editing methods (`slots`, `supercell`, `defects`, `rotate`, `flip`, `functionalize`) are thin delegates into `editing.py`.
- **`framework_io.py`** â€” versioned JSON (de)serialization of Framework objects (`Framework.save`/`load`): persists what CIF loses â€” the bond graph with orders, UFF4MOF types, anchor tags, and slot/SBU provenance â€” so a framework saved in one session stays editable in another.
- **`porosity.py`** â€” geometric porosity descriptors behind `Framework.void_fraction`/`largest_cavity_diameter`/`pore_limiting_diameter`: a periodic vdW-surface distance grid; PLD by binary search over probe radius with a wrap-detecting union-find (a percolating channel = an open component connected to its own periodic image).
- **`net.py`** â€” labeled quotient graphs (one node per slot/unit, edges carry periodic-image voltages). Verification (`verify_net`: exact edge-multiset comparison, possible because built atoms record slot provenance) and identification (`identify_net`: coordination-sequence signatures, caps pruned / 2-c vertices contracted, matched two-tier â€” uncontracted first, so a net beats its edge-decorated derivatives; a degree-multiset prefilter reads `raw_items()` and scans the full library without materializing it).
- **`deconstruct.py`** â€” the inverse pipeline. CIF/Structure â†’ EconNN bonds capped at `BOND_LENGTH_SLACK` Ă— UFF radius sums (EconNN is adaptive and bonds isolated guests to anything) â†’ 0-periodic components removed as guests â†’ clustering â†’ dummy-capped `Fragment`s (X at cut-bond midpoints) â†’ quotient graph (built directly from the cut list: one voltage-labeled edge per cut bond, so parallel bonds through different periodic images all survive; gauge shared with `autografs.net`) â†’ `identify_net`. Guest removal iterates until the bond graph is guest-free, so the fold and the subframework split come from one final graph. Clustering dispatches on metal presence: **metal-oxo** for MOFs (metals + C-free metal-bound atoms + carboxylate/sulfonate/phosphonate groups â€” cuts at carboxylate-Câ†”backbone-C and metalâ†”donor bonds), **branch-point** for metal-free COFs (`_organic_units`: ring systems + non-ring atoms collapse to super-vertices via `_ring_bonds` â€” a bounded zero-voltage cycle search on the periodic graph, no global unwrap â€” and external-degree â‰Ą3/2/1 â†’ node/linker/cap; single-node convention). Rod (1-periodic) units are detected and reported as `RodUnit`s (axis + repeat from the rank-1 mismatch lattice) and enter the quotient as points of extension joined along the axis (O'Keeffe PoE convention; typically a contracted-tier match â€” no edge centers); bare rods with no cuts and 2-periodic (layer) units raise `DeconstructionError`. Interpenetrated structures split into per-periodic-component quotient graphs, each identified independently (`subframework_nets`); `net_candidates` is their consensus, `n_periodic_components` the fold. Entry points: `Autografs.deconstruct(source)`, module-level `deconstruct(source, topologies=...)`.
- **`fingerprint.py`** â€” canonical assembly fingerprints: `AssemblyFingerprint` = (nets, block multiset gcd-reduced, fold), hashable, so "is this (net, SBU) combination realized?" is a set lookup. `from_deconstruction(result, library=...)` re-expresses blocks in a harvest's vocabulary via `deconstruct.match_fragment` (the read-only cousin of `merge_fragment`; unmatched blocks get an `unmatched:` marker that never collides); `from_framework(framework)` reads per-slot `sbu` provenance directly â€” the two meet when frameworks are built *from* the harvested library. Rod deconstructions have no finite block multiset and raise ValueError.
- **`harvest.py`** â€” batch SBU harvesting: runs `deconstruct` over many structures (directory/glob/iterable) and merges building units into one cross-structure-deduplicated `Fragment` library (`merge_fragment` from `deconstruct.py`), keeping per-fragment provenance, per-source net candidates, and a non-aborting failure report. `HarvestResult.building_units` excludes monotopic `cap` fragments (bound solvent). Rods dedupe separately into `HarvestResult.rods`/`rod_provenance` via `rods.merge_rod`. Entry: `Autografs.harvest(sources)`.
- **`rods.py`** â€” canonical rod identity behind harvest rod dedup (rod Stage B, #91): `canonical_rod(structure, rod_unit)` â†’ `RodRepeat`, one *chemical* repeat in a cylindrical frame (axis line through the perpendicular centroid â€” the only transverse origin a screw symmetry allows). Screw detection by self-matching under (axial shift, rotation): crystallographic repeat = screw_order Ă— chemical repeat (pitfall 3), falls back to no reduction when the cut pattern breaks the atoms' screw symmetry. `matches` compares modulo rotation Ă— phase Ă— proper flip (Hungarian per element in the cylindrical metric); **signed** screw angle is part of the identity â€” it is frame-invariant once the axial step is normalized positive, and enantiomeric helices must not merge (proper flips preserve helicity; don't "fix" this to abs()).
- **`editing.py`** â€” post-build editing on the bond graph: exact supercells (per-edge periodic image shift recovered by min-image rounding of unwrapped coords), statistical defects (seeded SBU removal + capping of dangling anchors), placed-SBU rotation/flip (isometries that fix the anchor atoms), framework functionalization (terminal-atom replacement by X-tagged groups). All ops return new Frameworks.
- **`charges.py`** â€” partial charges behind `Framework.assign_charges`: EQeq (Wilmer et al. 2012) as a numpy port validated against the reference C++ (element tables in `data/eqeq.py`; the exotic "Ewald" convention â€” erfc(R/Î·)-damped real sum + fixed-range reciprocal sum, Î·=50 Ă… â€” is reproduced literally, don't "fix" it). `CHARGE_METHODS` registry for pluggable schemes. Charges live on graph nodes (`charge` attr, `charge_method` graph attr), survive `framework_io` and editing, and flow into exports: CIF `_atom_site_charge` (P1 only), ASE initial charges, GULP (drops `noelectrostatics`).
- **`relax.py`** â€” LAMMPS relaxation backend behind `Framework.relax`. Shared staging helpers (`_write_lammps_inputs`, `_launch_lammps`) also serve `elastic.py`.
- **`ase_relax.py`** â€” ASE-calculator bridge behind `Framework.relax(calculator=...)`: any ASE calculator (or by name: "gfn-ff" via xtb-python, "gfn1" via tblite, "dftb"; "gfn2" rejected â€” no periodic implementation) drives a FrechetCellFilter+FIRE relaxation; bond graph preserved (order-canonical rebuild, #145), eV converted to the kcal/mol `Framework.energy` convention. All backends optional, import-guarded with install hints.
- **`elastic.py`** â€” elastic constants behind `Framework.elastic_properties`: stress-strain central differences on the in-process LAMMPS session (LAMMPS ELASTIC-example protocol: relax, then per-Voigt-strain `change_box` + fixed-cell FIRE), point-group projection of the tensor (averaged over cartesian rotations from `SpacegroupAnalyzer`), VRH moduli + directional Young's extrema on `ElasticProperties`.
- **`twist.py`** â€” commensurate twist search + moirĂ© bilayers behind `Framework.stack(mode="twisted", angle=...)`: plane-group-general CSL enumeration (coincidence vector pairs clustered by angle; reproduces the hexagonal closed form and the square `2Â·atan(n/m)` family â€” the tests use those as independent oracles), strained-commensurate mode (`max_strain` absorbs mismatch into the top layer, reported), snap-to-nearest with `angle_tolerance`, `max_atoms` guardrail (supercell ~1/Î¸Â˛). Family completeness is bounded by `max_index` â€” a family is only found once *two* independent coincidence vectors fit the index range. Exact angle/strain recorded as `twist_angle`/`twist_strain` graph attrs.
- **`plane_groups.py`** â€” the 17 plane groups, used for 2D layer nets.
- **`cgd.py`** â€” CGD parser and `autografs-topologies` CLI (RCSR download, spacegroup expansion, orbit extraction; `--use_iza` fetches + converts the IZA zeolite CIFs via `fetch.py` + `extract_topology.py`, entries under official uppercase codes).
- **`fetch.py`** â€” polite local fetching of externally-licensed sources (IZA now, EPINET later): license-acceptance gate (`require_acceptance`, interactive or `--accept-licenses`; nothing restricted ships in the wheel), resumable atomic cache (`~/.autografs/cache/<source>`), identifying UA, fixed request delay. Code list + filename convention (prefixes `-`/`*` stripped server-side) in `data/iza_codes.py`, shared with `scripts/make_iza_aliases.py`.
- **`extract_topology.py`** â€” `topology_from_tetrahedral(structure, name)`: idealized zeolite crystal (T atoms + bridging O, e.g. IZA CIFs) â†’ buildable Topology. Emits the exact CGD conventions (Z-encodes coordination on centers, X dummies at Tâ€“O midpoints = quarter points) and reuses `cgd.analyze`/`orbit_equivalence_classes`, so extracted zeolites are indistinguishable from CGD-imported nets down to the exact identification tier. Rejects interrupted/non-tetrahedral input (`TopologyExtractionError`). `topologies_from_tetrahedral_cifs` batches it with per-entry error accounting â€” the `--use_iza` conversion step.
- **`cli.py`** â€” `autografs` interactive wizard (questionary + rich). Deliberately has no non-interactive build flags; scripted use is the Python API.
- **`utils.py`** â€” XYZ parsing, UFF4MOF atom typing (constants in `data/uff4mof.py`), Fragmentâ†’graph conversions, GULP export.
- **`exceptions.py`** â€” `AutografsError` hierarchy: `AlignmentError`, `OverlapError`, `StackingError`, `RelaxationError`, `TopologyExtractionError`, `NetMismatchError`, `DeconstructionError`.

Pipeline: libraries â†’ geometric compatibility sieve (`list_building_units(sieve=...)` / `list_topologies(sieve=...)`) â†’ slot-type-to-SBU mappings â†’ per-slot alignment + cell optimization â†’ quality gates â†’ `Framework`.

## Gotchas

- **Everything ships in the wheel**: `data/topologies.json.gz` (2686 RCSR nets) and both SBU files are bundled; no generation step is needed. Regeneration (only when updating the library): `autografs-topologies --use_rcsr -o topologies.json.gz`. `scripts/cgd2pkl.py` is a deprecated shim for that command.
- **Research tooling lives in `scripts/`, not the library**: `scripts/benchmarks/` (net-ID accuracy, round-trip closure, throughput) and `scripts/funnel/` (staged multi-fidelity screening driver: level escalation asbuiltâ†’UFF4MOFâ†’GFN-FF/GFN1/DFTB+, top-N selection, per-candidate provenance, checkpoint/restart compatible with `build_all` checkpoint dirs, Spearman rank-stability report). Both have path-insertion smoke tests in `tests/` so CI catches API breakage.
- Dummy atoms use pymatgen's `"X"` species convention throughout. Slot/SBU compatibility is decided by dummy count + arm-direction match, NOT point group (symbols are metadata only).
- In `cgd.py`, `# EDGE_CENTER` lines in RCSR files look commented out but are deliberately uncommented by the parser (they create the 2-connected slots). Preserve that behavior when refactoring.
- `Autografs._validate_mappings` deep-copies fragments: alignment mutates them in place, and aliasing would corrupt the SBU library. Keep that invariant.
- Framework node ids must stay contiguous 0..n-1: `min_contact` and relax index pymatgen Structure sites by sorted node id. Any graph-editing operation must relabel before returning (see `editing._relabel`). Graph node coords are unwrapped cartesian; a bond's periodic image is implicit and recovered by min-image rounding â€” editing relies on it.
- Formatting/linting is ruff (not black/isort/flake8); mypy targets 3.12 even though the runtime floor is 3.11 (numpy stubs use PEP 695).
