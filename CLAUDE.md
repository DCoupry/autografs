# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

AuToGraFS generates Metal-Organic Frameworks (MOFs), COFs and other periodic crystalline structures by mapping molecular building blocks (SBUs) onto topological blueprints, and runs the pipeline in reverse (deconstruction). Python >= 3.11. Built on pymatgen, ASE, networkx, scipy. Docs: `README.md` is the overview + FAQ; feature guides live in `docs/*.md` (building, cofs-and-stacking, editing, deconstruction, cli, extending, internals, coverage). Keep them accurate to the current API тАФ when changing behaviour, update the relevant `docs/` page (and the README FAQ/Why bullets if the change is user-facing).

## Architecture

Modules in `src/autografs/`, in pipeline order:

- **`fragment.py`** тАФ `Fragment`: a pymatgen `Molecule` plus dummy-atom (`"X"`) connection points. Represents both SBUs and topology slots. Compatibility (`has_compatible_symmetry`) is geometric: directional RMSD between unit arm vectors (threshold `COMPATIBILITY_MAX_RMSD = 0.35`), not point-group symbols. Also pre-build editing: `rotate`, `flip`, `functionalize`.
- **`topology.py`** тАФ `Topology`: a pymatgen `Lattice` plus an array of `Fragment` slots. `mappings` groups slots by crystallographic orbit (`equivalence_class`); one SBU choice covers all slots of an orbit. `is_2d` marks layer nets (c is frozen slab padding).
- **`symmetry.py`** тАФ blueprint space-group operations, derived **numerically from the slot-centre set** (#174 groundwork): `Topology` records only the spacegroup *number*, and recovering operations from it would reintroduce a setting/origin ambiguity against the CGD embedding тАФ so `blueprint_operations` finds them from the embedding itself (metric-preserving integer rotation parts, entries in {-1,0,1} тАФ enough for every conventional setting; translations from a rarest-orbit reference slot; orbit labels preserved, improper ops included since mirrors constrain displacements too). `orbit_displacements` turns them into the symmetry-allowed slot-displacement parametrization the embedding-relaxation work needs: one displacement per orbit, restricted to the representative's site-symmetry fixed space (`bases`, metric-orthonormalized so optimizer steps mean ├Е), propagated to orbit members by the relating operation (`d_slot = W @ d_rep`). Key library fact: pcu/dia/srs/nbo/fcu/sql/hcb are **fully pinned** (`n_free == 0` тАФ exactly why those nets build right today), while sra 4, etb 2, etb-e 5, tbo 3, pts 2, sod 1 carry the free proportions #174 measures. ~0.2тАУ0.4 s per net, tbo's 152 slots included. `LazyTopologyLibrary` materializes entries on first access (don't `dict.update()` it тАФ that materializes everything). Lookup-only aliases via `attach_aliases` (the builder attaches `data/iza_aliases.json`, 55 IZA zeolite codes тЖТ lowercase RCSR names; aliases resolve in `[]`/`in` but never appear in iteration). Legacy dill pickles still load with a warning.
- **`alignment.py`** тАФ the numpy build core. Directional matching (Hungarian + Kabsch, proper rotations only тАФ chirality preserved) and the cell objective (RMS deviation of anchor-pair distances from Cordero covalent bond lengths, over the crystal system's free parameters only). Everything precomputed into a `BuildPlan`; no pymatgen objects inside the optimization loop. **Embedding relaxation** (#174, `build(..., relax_embedding=True)`): the blueprint's symmetry-allowed slot displacements (`symmetry.orbit_displacements`) join the parameter vector after the cell block, and each pair's closure gap gains two **anchor-direction** terms тАФ the deviation of the bond vector from a bond-length step along each end's own anchorтЖТdummy direction (`SlotPlacement.bond_units`, distinct from the centroid arm for a bulky node). The two are co-dependent by measurement: dof without the term leaves a floppy net, and the term without dof is actively harmful тАФ on pinned nets it can only fight the cell against an unresolvable arm mismatch (measured on the CoRE MOF faithful set: bond residual 0.21тЖТ0.54 ├Е for a packing gain that isn't worth it) тАФ so **fully pinned nets keep the legacy objective exactly, flag or no flag** (`_shifts` returns None when `n_free == 0`). All three terms are lengths, so no weighting. Default off тАФ the legacy fixed-slot objective is preserved bit-for-bit when `slot_disp is None`.
- **`builder.py`** тАФ `Autografs`: entry point. Loads SBUs (`data/defaults.xyz` + `data/pormake.xyz` + user XYZ) and topologies (`data/topologies.json.gz`). `build(topology, mappings, max_rmsd=..., min_distance=...)` тЖТ `Framework`; gates raise `AlignmentError`/`OverlapError` instead of returning bad structures. `build_all()` enumerates combinations (seeded sampling above `max_per_topology`, `n_jobs` worker processes). Module-level `build_framework()` is the library-independent core used by workers.
- **`framework.py`** тАФ `Framework`: the result. The networkx bond graph is the source of truth (symbols, coords, bond orders, UFF4MOF types, tags, per-atom `slot`/`sbu` provenance); views/exports: `structure`, `to_ase`, `write_cif`, `to_gulp`, `min_contact`, `stack` (2D layer тЖТ COF crystal, AA/AB/serrated/staggered), `relax` (in-process LAMMPS/UFF4MOF, optional `[relax]` extra). Post-build editing methods (`slots`, `supercell`, `defects`, `rotate`, `flip`, `functionalize`) are thin delegates into `editing.py`.
- **`framework_io.py`** тАФ versioned JSON (de)serialization of Framework objects (`Framework.save`/`load`): persists what CIF loses тАФ the bond graph with orders, UFF4MOF types, anchor tags, and slot/SBU provenance тАФ so a framework saved in one session stays editable in another.
- **`porosity.py`** тАФ geometric porosity descriptors behind `Framework.void_fraction`/`largest_cavity_diameter`/`pore_limiting_diameter`: a periodic vdW-surface distance grid; PLD by binary search over probe radius with a wrap-detecting union-find (a percolating channel = an open component connected to its own periodic image).
- **`net.py`** тАФ labeled quotient graphs (one node per slot/unit, edges carry periodic-image voltages). Verification (`verify_net`: exact edge-multiset comparison, possible because built atoms record slot provenance; **rod** frameworks тАФ whose slots are repeats+linker placements, not blueprint slots тАФ instead compare the framework's uncontracted signature to the blueprint's PoE form `topology_rod_quotient_edges` over every axial/helical run, the gcd fold absorbing the `n_repeats` supercell, `build_rod(..., verify_net=True)` gating it, #158) and identification (`identify_net`: coordination-sequence signatures, caps pruned / 2-c vertices contracted, matched two-tier тАФ uncontracted first, so a net beats its edge-decorated derivatives; a degree-multiset prefilter reads `raw_items()` and scans the full library without materializing it). The prune/contract reduction is public as `contract_quotient_edges` тАФ `net_signature` routes through it, and it is what lets two embeddings of the same net be compared however they are decorated (`scripts/benchmarks/embedding.py`). `axial_runs` (rod Stage C2) finds a blueprint's *straight* slot runs тАФ cyclic quotient walks with every step displacement parallel and a nonzero closure voltage (pcu: three node+axial-edge chains; zigzag nets: none). `helical_runs` (#158) finds the *spiralling* runs straight detection misses тАФ a constant-axial-advance walk whose node slots fit a constant screw about a fixed axis line (`_fit_screw`): reports `HelicalRun` with signed `screw_angle`/`screw_order` (the blueprint mirror of a rod's own `RodRepeat` screw), so etb resolves to its 3_1 metal-oxo channels (order 3, ┬▒120┬░) and srs to its 4_1 (order 4, +90┬░); pcu/zigzag give none, and 2D layer nets are skipped (an in-plane zig-zag is no 3D rod channel).
- **`deconstruct.py`** тАФ the inverse pipeline. CIF/Structure тЖТ EconNN bonds capped at `BOND_LENGTH_SLACK` ├Ч UFF radius sums (EconNN is adaptive and bonds isolated guests to anything) тЖТ 0-periodic components removed as guests тЖТ clustering тЖТ dummy-capped `Fragment`s (X at cut-bond midpoints) тЖТ quotient graph (built directly from the cut list: one voltage-labeled edge per cut bond, so parallel bonds through different periodic images all survive; gauge shared with `autografs.net`) тЖТ `identify_net`. Guest removal iterates until the bond graph is guest-free, so the fold and the subframework split come from one final graph. Clustering dispatches on metal presence: **metal-oxo** for MOFs (metals + C-free metal-bound atoms + carboxylate/sulfonate/phosphonate groups тАФ cuts at carboxylate-CтЖФbackbone-C and metalтЖФdonor bonds), **branch-point** for metal-free COFs (`_organic_units`: ring systems + non-ring atoms collapse to super-vertices via `_ring_bonds` тАФ a bounded zero-voltage cycle search on the periodic graph, no global unwrap тАФ and external-degree тЙе3/2/1 тЖТ node/linker/cap; single-node convention). Rod (1-periodic) units are detected and reported as `RodUnit`s (axis + repeat from the rank-1 mismatch lattice) and enter the quotient as points of extension joined along the axis (O'Keeffe PoE convention; typically a contracted-tier match тАФ no edge centers); bare rods with no cuts and 2-periodic (layer) units raise `DeconstructionError`. Interpenetrated structures split into per-periodic-component quotient graphs, each identified independently (`subframework_nets`); `net_candidates` is their consensus, `n_periodic_components` the fold. Entry points: `Autografs.deconstruct(source)`, module-level `deconstruct(source, topologies=...)`.
- **`fingerprint.py`** тАФ canonical assembly fingerprints: `AssemblyFingerprint` = (nets, block multiset gcd-reduced, fold), hashable, so "is this (net, SBU) combination realized?" is a set lookup. `from_deconstruction(result, library=...)` re-expresses blocks in a harvest's vocabulary via `deconstruct.match_fragment` (the read-only cousin of `merge_fragment`; unmatched blocks get an `unmatched:` marker that never collides); `from_framework(framework)` reads per-slot `sbu` provenance directly тАФ the two meet when frameworks are built *from* the harvested library. Rod deconstructions have no finite block multiset and raise ValueError.
- **`harvest.py`** тАФ batch SBU harvesting: runs `deconstruct` over many structures (directory/glob/iterable) and merges building units into one cross-structure-deduplicated `Fragment` library (`merge_fragment` from `deconstruct.py`), keeping per-fragment provenance, per-source net candidates, and a non-aborting failure report. `HarvestResult.building_units` excludes monotopic `cap` fragments (bound solvent). Rods dedupe separately into `HarvestResult.rods`/`rod_provenance` via `rods.merge_rod`. Entry: `Autografs.harvest(sources)`.
- **`rods.py`** тАФ canonical rod identity behind harvest rod dedup (rod Stage B, #91): `canonical_rod(structure, rod_unit)` тЖТ `RodRepeat`, one *chemical* repeat in a cylindrical frame (axis line through the perpendicular centroid тАФ the only transverse origin a screw symmetry allows). Screw detection by self-matching under (axial shift, rotation): crystallographic repeat = screw_order ├Ч chemical repeat (pitfall 3), falls back to no reduction when the cut pattern breaks the atoms' screw symmetry. `matches` compares modulo rotation ├Ч phase ├Ч proper flip (Hungarian per element in the cylindrical metric); **signed** screw angle is part of the identity тАФ it is frame-invariant once the axial step is normalized positive, and enantiomeric helices must not merge (proper flips preserve helicity; don't "fix" this to abs()). Stage C1: `rod_fragment` тЖТ `RodFragment` (the buildable form: local-frame atom template + connection arms from `RodUnit.cut_vectors` + internal bond graph from `RodUnit.internal_bonds`, both rotated/mapped into the cylindrical template frame). `_local_positions` unwraps the rod by walking its bond graph **intra-cell bonds first** (periodic-closure bonds last), so a screw rod lays out monotonically along its axis тАФ the chemical-repeat reduction then keeps naturally-bonded atoms together and the **screw-aware** slab correspondence (de-screw azimuth by slab) records correct template bonds for **helical** rods too, not just straight (#158). `STRAIGHT_SCREW_TOL` still gates forward *building* (`build_rod` rejects helical placement тАФ that's the next #158 increment), but the fragment itself is now complete for helical rods. Versioned JSON sidecar via `save_rods`/`load_rods` (`HarvestResult.write_rods`) тАФ rods can't join the XYZ SBU format.
- **`rod_build.py`** тАФ forward building of rod frameworks, straight **and helical** (rod Stage C) behind `Autografs.build_rod` / `build_rod_framework(topology, rod, linkers)`: places a `RodFragment` on a blueprint's slot run + finite SBUs on every other slot. Two run kinds: a **straight** run (`net.axial_runs`, one PoE node/period, supercelled `n_repeats` up) or a **helical** run (`net.helical_runs`, `screw_order` node slots filled 1:1 by the rod's repeats тАФ no supercell; `_select_run` matches the rod's signed screw, #158). The rod **pins** the run-axis cell parameter to `n_repeats ├Ч chemical repeat` (removed from the free set; `n_repeats = max(2, screw_order)` тАФ enough to close the screw, тЙе2 so the continuation bond is a distinct node pair, not a parallel edge тАФ pitfall 9); placement lays down `n_repeats` copies by the **screw operation** (copy *n* rotated `n ├Ч screw_angle` about the axis тАФ about the **run's axis line** `axis_point` for a helical run, the node itself for straight; linkers spiralling with it тАФ pure translation for a straight rod), optimizing (in-plane scale, axis rotation, axial phase) against covalent anchor-length targets (pitfall 10); a second **relief** pass rotates each ditopic linker about its own arm axis (keeps anchors fixed тЖТ bonds stay closed) to maximize the minimum inter-unit contact, one shared ╧Ж per orbit (un-clashes coplanar rings тАФ the finite pipeline leaves that to manual `rotate()`). Rod-linker bonds are **explicit graph edges** (tags assume one connection/atom; a rod anchor carries several), all atoms tag 0. Closure + RMSD + `min_contact` gates. **Cross-linked multi-rod nets** (`etb` / MOF-74 proper, #158): `_select_runs` returns **every** matching helical run (etb: six interleaved 3_1 helices, three of each hand); `_RodBuild` holds one `rod_specs` entry per run and places a rod on each тАФ reflecting the template/arms (local yтЖТтИТy) for opposite-handedness runs since etb is centrosymmetric тАФ with the linkers on the slots no run claims (they bridge *different* rods). The reference node per run is **unwrapped** onto its own cylinder (`_unwrapped_node_frac`; a home-cell image can be a perp-wrapped copy off the axis, breaking radius/azimuth). `net._verify_rod_net` groups helical runs by `screw_order` and contracts each group's edge-centers (`topology_rod_quotient_edges` takes a run list), so etb's full PoE form is checked. **Mixed rod/finite mappings** (#168): `linkers` is a Fragment *or* a `{slot type | slot index: SBU}` mapping (`_validate_linkers`, covering the lateral slots only тАФ the run is the rod's), lateral SBUs may be **polytopic**, and lateral arm counts come from **run membership** (`_slot_adjacency`: node degree тИТ the connections the run consumes) rather than the old `degree тИТ 2`. Placement reuses `alignment.match_directions` + `kabsch` (Hungarian arm assignment, re-solved once at the optimized cell by `refresh_assignments` тАФ a polytopic slot's arm directions move with the anisotropic cell); relief ╧Ж applies only to species with a genuine arm axis (2 near-antiparallel arms). The old bipartite rod-armтЖФlinker-anchor Hungarian is replaced by `_pair_ports`: **every** connection point (rod arm tip + every SBU dummy) is paired, since no library net with a rod run has polytopic laterals bonded only to run nodes тАФ linkerтЖФlinker bonds are unavoidable. *Which* slots may bond comes from the blueprint, not geometry: `_port_budget` reads the rod-form quotient (run edge centers + emptied slots already contracted, the rods' own continuation edges excluded) into one bond count per slot pair, and greedy min-image distance only chooses which ports/images realize them. Pairing by pure proximity was tried first and is **wrong**: the objective over the cell scale goes multi-modal and its global minimum can be a spurious pairing at a badly inflated cell (found on real CAXVOO тАФ 22 ├Е cell at obj 0.13 beating the true branch). Budgets also make the matching complete for free (each slot's budgets sum to its port count) and make the built edge multiset equal the blueprint's by construction. **Empty (contracted) slots**: a lateral slot mapped to `None` places nothing and bonds its two neighbours directly тАФ what real rod MOFs need, since a MOF-74 rod binds straight onto the 4-c DOBDC while the blueprint decorates that edge (the same reason such structures identify on the contracted tier). Only 2-connected slots may be emptied; the build records them as the `rod_empty_slots` graph marker (propagated by `framework_io`/`replicated_graph` beside `rod_build`), and `topology_rod_quotient_edges(topology, runs, empty_slots)` contracts them so `verify_net` still compares like with like. Validated end-to-end: synthetic 2_1 helix on pcu; **4_1 тАУZnтАУOтАУ helix on `unc`** (single-helix, builds clean, `verify_net(unc)` passes, round-trips to net `unc` + a screw-order-4/+90┬░ rod); **six cross-linked 3_1 rods on `etb`** (`verify_net(etb)` passes, `identify_net` тЖТ `etb`, both handednesses placed; the small cell crowds the synthetic fixture so the test relaxes `min_distance` тАФ `relax()` cleans it); and **`etb-e` with two lateral species** (36 ditopic + 9 tetratopic laterals, 36 linkerтЖФlinker bonds, `verify_net` + `identify_net` тЖТ `etb-e`, and it clears the *default* contact gate unrelaxed тАФ the MOF-74 bonding as it really is), plus the same net with the decorations **empty**. Real-CIF (local CoRE MOF, not in CI): a harvested CAXVOO Fe 3_1 rod + its real 4-c DOBDC builds on `etb-e` with the decorations empty тАФ right composition, rod-axis parameter exact (6.911 ├Е), `verify_net` + `identify_net` тЖТ `etb-e` тАФ but the *idealized* embedding's proportions are off for the real chemistry (ideal nodeтЖТlinker separation 0.168 of the in-plane cell vs 0.284 measured), so the packing is compressed (a 22 ├Е vs 26 ├Е) and clashes until relaxed. That gap is the embedding, not the pipeline. **Woven multi-axis packings** (#168 item 2): runs group into **axis families** (`_RodBuild.families`), each pinning its own cell parameter and carrying its own `(theta, z0)` тАФ the parameter vector is `[scale, theta_f, z0_f, ...]` (`unpack`), with the coarse rotation grid run per family (a product grid would be 16^families). A fully woven packing pins all three axes, so the cell is determined outright and `scale` has nothing left to vary. Axis lines are stored **fractional** (`axis_point_frac`) since a scalar scale no longer describes an arbitrarily-pinned cell. `_select_runs` groups candidate runs by **period**, not axis: one rod pins every axis it runs along to the same length, so only equal-period runs can be woven (`bbe` spirals 2_1 along a and b with *different* periods тЖТ only the busiest set is built, the rest stay lateral). Validated on **`bmn`** (six 4_1 helices, two per cubic axis, covering all 24 node slots with no overlap; cubic cell = 4 x chemical repeat on every axis, `verify_net` + `identify_net` тЖТ `bmn`, contact gate clean). **Multi-PoE straight runs** (#168): a straight run may chain **several** PoE nodes per period (66 library nets do), each holding one chemical repeat тАФ so the period already contains k of them and `_repeat_counts(nodes_per_period, screw_order)` decides how many periods to stack (`copies`) and how many repeats that is; `linker_copies = copies`, `_cell_one_period` spans `k ├Ч period`, and `repeat_slot` tiles the run's nodes in **axial** order. Two gates, both measured by `_run_nodes`: the nodes must be **evenly spaced** (a rod's repeats are) and related by a **constant rotation** whose value the rod's own screw must match (`_screw_fits`). That rotation is the subtle part тАФ a run can be perfectly straight (centres collinear) while the slots' *orientations* turn, and since the centres sit **on** the axis a positional screw fit is degenerate, so it is measured from the nodes' own lateral **arm azimuths** instead. `cds` turns 90┬░/node and wants a 4_1 rod; `btu` is unevenly spaced and `cdl` turns 0/90/0/90 тАФ both correctly refused. `_select_runs` offers a rod the blueprint's other runs when it does not fit the multi-node one (a screwless rod takes one of `cds`'s one-node runs). Validated on **`cds`** (two 4-c nodes/period 90┬░ apart, 4_1 rod, 4 repeats over 2 stacked periods, `verify_net` + `identify_net` тЖТ `cds`, contact gate clean). **Axis-multiple runs** (#173): a helical channel may close only after several cells (twt advances c/3 and turns 60┬░/node, so a c translation lands on a half-turn-rotated copy and the run direction is тЯи002тЯй). 13 library nets carry one тАФ тЯи002тЯй fne/fnt/mjd/nlr/qrf/src/twt/twt-e/uom/uoo, тЯи003тЯй hqx/mdf/yan тАФ and for 7 (fne, fnt, src, twt, twt-e, uom, uoo) it is the channel's *only* description, so those were unbuildable before. Still axis-parallel, so no new parametrization тАФ `_run_axis(run)` returns `(axis index, multiple)` and the pin is `axis_length / multiple` (per **family**, since a woven build could mix multiples; `_cell_one_period` divides too). The picture: one cell then holds *all* the run's node slots as `multiple` **interleaved passes** of the same rod, not one pass тАФ the passes differ by a lattice translation in height but *not* in azimuth, so they are not periodic images and the repeats stay distinct. `_select_runs`' node-set dedup already prefers a net's single-cell description of the *same* channel (mjd/nlr/qrf/yan report both walks over the same node slots), so those are unchanged; where the two are different channels (mdf's тЯи003тЯй helices share no nodes with its тЯи001тЯй ones) the busiest-set/shortest-period tie-break decides and `run=` forces the other. Validated on **`twt`** (6_1 rod, тЯи002тЯй, `verify_net` + `identify_net` тЖТ `twt`, contact gate clean, c = 6 ├Ч repeat ├╖ 2); `fnt`/`uom`/`uoo` also build and verify. A *straight* run with a multiple is **refused**, not built: none exists in the library (every axis-multiple run is helical) and the straight path supercells whole blueprint periods. **Diagonal runs** (#173, second half): 112 nets run a channel down a lattice **diagonal** (тЯи011тЯй mostly, then тЯи111тЯй/тЯи112тЯй/тЯи012тЯй; bcu chains along its body diagonals). The pinned quantity `|direction @ cell|` is a *combination* of cell parameters, so it cannot be substituted into a row тАФ instead `cell()` **stretches** the whole cell along the run's own direction by `pin_scale = axis_length / |direction @ blueprint|` and by `scale` perpendicular (`cell*scale + outer(blueprint @ n╠В, n╠В)*(pin_scale тИТ scale)`). Two invariants the rest of the pipeline needs hold at *every* scale: the period is exactly the rod's length (the period vector is тИе n╠В, so only `pin_scale` acts on it) and the run **direction does not move**, keeping the transverse frame/screw axis/arm embedding scale-invariant. The cell's symmetry drops (bcu goes rhombohedral) тАФ correct, a rod down one diagonal singles it out. Families are keyed by **direction** now, not axis index, and `_run_axis` returns `(None, gcd)` for a diagonal. **The one real bug found**: `initial_guess`'s perpendicular-distance heuristic measures a *home-cell* separation, which for a diagonal is not the bonded one тАФ fcu started at 2.1 against a true 5.9 and Nelder-Mead never left that basin (4/19 nets built). For a diagonal, `scale0 = pin_scale` instead (isotropic = the blueprint's own proportions at the rod's size) тЖТ **13/19**. All diagonal runs are **straight** (`helical_runs` walks one cell axis by construction) and 91% need no supercell; the rest are refused тАФ stacking along a diagonal is a sublattice **transformation**, not a row scale. Axis + diagonal families cannot be mixed. Validated on **`bcu`** (тЯи111тЯй, 2 nodes/period, 6 ditopic laterals, `verify_net` + `identify_net` тЖТ `bcu`, contact clean, cell rhombohedral); bct/bcu-x/fcu/ilc/ile/kag/lhb/reo/rob/sxb/xak/bcg also build+verify. Of the 6 that do not: 2 (ild, roa) close but realize a *different* net тАФ greedy pairing picking images the blueprint does not, which is what `verify_net=True` catches тАФ and 4 miss closure by 0.4тАУ0.6 ├Е with one free transverse scale.
- **`editing.py`** тАФ post-build editing on the bond graph: exact supercells (per-edge periodic image shift recovered by min-image rounding of unwrapped coords), statistical defects (seeded SBU removal + capping of dangling anchors), placed-SBU rotation/flip (isometries that fix the anchor atoms), framework functionalization (terminal-atom replacement by X-tagged groups). All ops return new Frameworks. The tag/anchor-based ops (`defects`/`rotate`/`flip`/`functionalize`) refuse rod (`build_rod`) frameworks via `_reject_rod` (rod Stage C4, #91): rods carry explicit inter-unit bonds and no anchor tags, and a rod has no finite SBU to remove/reorient (pitfall 16); `Framework.is_rod` reads the `rod_build` graph marker, which `replicated_graph` and `framework_io` propagate so supercell and save/load keep the guard.
- **`charges.py`** тАФ partial charges behind `Framework.assign_charges`: EQeq (Wilmer et al. 2012) as a numpy port validated against the reference C++ (element tables in `data/eqeq.py`; the exotic "Ewald" convention тАФ erfc(R/╬╖)-damped real sum + fixed-range reciprocal sum, ╬╖=50 ├Е тАФ is reproduced literally, don't "fix" it). `CHARGE_METHODS` registry for pluggable schemes. Charges live on graph nodes (`charge` attr, `charge_method` graph attr), survive `framework_io` and editing, and flow into exports: CIF `_atom_site_charge` (P1 only), ASE initial charges, GULP (drops `noelectrostatics`).
- **`relax.py`** тАФ LAMMPS relaxation backend behind `Framework.relax`. Shared staging helpers (`_write_lammps_inputs`, `_launch_lammps`) also serve `elastic.py`.
- **`ase_relax.py`** тАФ ASE-calculator bridge behind `Framework.relax(calculator=...)`: any ASE calculator (or by name: "gfn-ff" via xtb-python, "gfn1" via tblite, "dftb"; "gfn2" rejected тАФ no periodic implementation) drives a FrechetCellFilter+FIRE relaxation; bond graph preserved (order-canonical rebuild, #145), eV converted to the kcal/mol `Framework.energy` convention. All backends optional, import-guarded with install hints.
- **`elastic.py`** тАФ elastic constants behind `Framework.elastic_properties`: stress-strain central differences on the in-process LAMMPS session (LAMMPS ELASTIC-example protocol: relax, then per-Voigt-strain `change_box` + fixed-cell FIRE), point-group projection of the tensor (averaged over cartesian rotations from `SpacegroupAnalyzer`), VRH moduli + directional Young's extrema on `ElasticProperties`.
- **`twist.py`** тАФ commensurate twist search + moir├й bilayers behind `Framework.stack(mode="twisted", angle=...)`: plane-group-general CSL enumeration (coincidence vector pairs clustered by angle; reproduces the hexagonal closed form and the square `2┬╖atan(n/m)` family тАФ the tests use those as independent oracles), strained-commensurate mode (`max_strain` absorbs mismatch into the top layer, reported), snap-to-nearest with `angle_tolerance`, `max_atoms` guardrail (supercell ~1/╬╕┬▓). Family completeness is bounded by `max_index` тАФ a family is only found once *two* independent coincidence vectors fit the index range. Exact angle/strain recorded as `twist_angle`/`twist_strain` graph attrs.
- **`plane_groups.py`** тАФ the 17 plane groups, used for 2D layer nets.
- **`cgd.py`** тАФ CGD parser and `autografs-topologies` CLI (RCSR download, spacegroup expansion, orbit extraction; `--use_iza` fetches + converts the IZA zeolite CIFs via `fetch.py` + `extract_topology.py`, entries under official uppercase codes).
- **`fetch.py`** тАФ polite local fetching of externally-licensed sources (IZA now, EPINET later): license-acceptance gate (`require_acceptance`, interactive or `--accept-licenses`; nothing restricted ships in the wheel), resumable atomic cache (`~/.autografs/cache/<source>`), identifying UA, fixed request delay. Code list + filename convention (prefixes `-`/`*` stripped server-side) in `data/iza_codes.py`, shared with `scripts/make_iza_aliases.py`.
- **`extract_topology.py`** тАФ `topology_from_tetrahedral(structure, name)`: idealized zeolite crystal (T atoms + bridging O, e.g. IZA CIFs) тЖТ buildable Topology. Emits the exact CGD conventions (Z-encodes coordination on centers, X dummies at TтАУO midpoints = quarter points) and reuses `cgd.analyze`/`orbit_equivalence_classes`, so extracted zeolites are indistinguishable from CGD-imported nets down to the exact identification tier. Rejects interrupted/non-tetrahedral input (`TopologyExtractionError`). `topologies_from_tetrahedral_cifs` batches it with per-entry error accounting тАФ the `--use_iza` conversion step.
- **`cli.py`** тАФ `autografs` interactive wizard (questionary + rich). Deliberately has no non-interactive build flags; scripted use is the Python API.
- **`utils.py`** тАФ XYZ parsing, UFF4MOF atom typing (constants in `data/uff4mof.py`), FragmentтЖТgraph conversions, GULP export.
- **`exceptions.py`** тАФ `AutografsError` hierarchy: `AlignmentError`, `OverlapError`, `StackingError`, `RelaxationError`, `TopologyExtractionError`, `NetMismatchError`, `DeconstructionError`.

Pipeline: libraries тЖТ geometric compatibility sieve (`list_building_units(sieve=...)` / `list_topologies(sieve=...)`) тЖТ slot-type-to-SBU mappings тЖТ per-slot alignment + cell optimization тЖТ quality gates тЖТ `Framework`.

## Gotchas

- **Everything ships in the wheel**: `data/topologies.json.gz` (2686 RCSR nets) and both SBU files are bundled; no generation step is needed. Regeneration (only when updating the library): `autografs-topologies --use_rcsr -o topologies.json.gz`. `scripts/cgd2pkl.py` is a deprecated shim for that command.
- **Research tooling lives in `scripts/`, not the library**: `scripts/benchmarks/` (net-ID accuracy, round-trip closure, throughput, and `embedding.py` тАФ the #174 metric: idealized-vs-real embedding proportions, scored as a scale-free *reduced edge length* `d / (fold┬╖V/n)^(1/3)` on the contracted quotient graph of both sides, split into a **size** error that predicts the built cell volume ratio without building and a **shape** error one scale cannot fix; summarized over the *buildable* population тАФ a structure whose units no slot accepts fails for an unrelated reason тАФ and layer nets excluded, their slab padding making V arbitrary), `scripts/funnel/` (staged multi-fidelity screening driver: level escalation asbuiltтЖТUFF4MOFтЖТGFN-FF/GFN1/DFTB+, top-N selection, per-candidate provenance, checkpoint/restart compatible with `build_all` checkpoint dirs, Spearman rank-stability report), and `scripts/rods/scan_rod_mofs.py` (deconstruct a directory of CIFs, report rod building units with canonical screw/net; `report --elements` palette filter is **optional, off by default** тАФ scans all; reads a user directory, redistributes nothing). All have path-insertion smoke tests in `tests/` so CI catches API breakage.
- Dummy atoms use pymatgen's `"X"` species convention throughout. Slot/SBU compatibility is decided by dummy count + arm-direction match, NOT point group (symbols are metadata only).
- In `cgd.py`, `# EDGE_CENTER` lines in RCSR files look commented out but are deliberately uncommented by the parser (they create the 2-connected slots). Preserve that behavior when refactoring.
- `Autografs._validate_mappings` deep-copies fragments: alignment mutates them in place, and aliasing would corrupt the SBU library. Keep that invariant.
- Framework node ids must stay contiguous 0..n-1: `min_contact` and relax index pymatgen Structure sites by sorted node id. Any graph-editing operation must relabel before returning (see `editing._relabel`). Graph node coords are unwrapped cartesian; a bond's periodic image is implicit and recovered by min-image rounding тАФ editing relies on it.
- Formatting/linting is ruff (not black/isort/flake8); mypy targets 3.12 even though the runtime floor is 3.11 (numpy stubs use PEP 695).
